# Mango Business Management

A complete business management application for mango trading businesses. Track sales, stock purchases, customers, suppliers, profitability, and more.

## Tech Stack

- **Frontend:** React 18 + Vite
- **Styling:** Tailwind CSS v3
- **Backend & Auth:** Supabase (PostgreSQL, Auth, Edge Functions)
- **Charts:** Recharts
- **Export:** SheetJS (Excel), jsPDF (PDF)
- **Hosting:** Netlify

## Setup Instructions

### 1. Create a Supabase Project

1. Go to [supabase.com](https://supabase.com) and create a free account
2. Click **New Project** and fill in the details
3. Wait for your project to finish setting up

### 2. Get Your API Credentials

1. In your Supabase dashboard, go to **Project Settings → API**
2. Copy the **Project URL** (e.g., `https://xxxx.supabase.co`)
3. Copy the **anon public** key

### 3. Run the Database Schema

1. In your Supabase dashboard, go to **SQL Editor**
2. Click **New Query**
3. Open the file `supabase/schema.sql` from this project
4. Copy the entire contents and paste it into the SQL Editor
5. Click **Run** — all tables, policies, indexes, and seed data will be created

### 4. Configure the Frontend

1. Open `src/supabaseClient.js`
2. Replace `SUPABASE_URL` with your Project URL
3. Replace `SUPABASE_ANON_KEY` with your anon public key

```js
const SUPABASE_URL = 'https://your-project-id.supabase.co';
const SUPABASE_ANON_KEY = 'your-anon-key-here';
```

### 5. Create Your Owner Account

1. In your Supabase dashboard, go to **Authentication → Users**
2. Click **Add User → Create New User**
3. Enter your email and a strong password
4. Click **Create User**
5. Now go to **SQL Editor** and run this exact SQL to promote your account to owner:

```sql
UPDATE public.profiles
SET role = 'owner', full_name = 'Your Name'
WHERE id = (SELECT id FROM auth.users WHERE email = 'your-email@example.com');
```

Replace `'Your Name'` and `'your-email@example.com'` with your actual name and the email you used.

### 6. Deploy the Edge Function (for User Management)

1. Install the [Supabase CLI](https://supabase.com/docs/guides/cli)
2. Log in to the CLI:
   ```bash
   supabase login
   ```
3. Link your project:
   ```bash
   supabase link --project-ref your-project-ref
   ```
   (Find your project ref in **Project Settings → General**)
4. Deploy the function:
   ```bash
   supabase functions deploy create-user --no-verify-jwt
   ```

> **Note:** The Edge Function uses the `SUPABASE_SERVICE_ROLE_KEY` which is automatically available as an environment variable in Edge Functions. This key is never exposed to the frontend client.

### 7. Install Dependencies & Build

```bash
npm install
npm run build
```

To run locally for development:

```bash
npm run dev
```

### 8. Deploy to Netlify

**Option A — Drag and Drop:**
1. Go to [app.netlify.com/drop](https://app.netlify.com/drop)
2. Drag and drop the `dist` folder generated by `npm run build`
3. Your site is live!

**Option B — Connect Git Repository:**
1. Push this project to a GitHub/GitLab repository
2. In Netlify, click **Add New Site → Import an existing project**
3. Connect your repository
4. Set build command: `npm run build`
5. Set publish directory: `dist`
6. Deploy

### 9. Log In

Open your deployed URL (or `http://localhost:5173` for local dev) and log in with the owner account you created in Step 5.

## Features

### All Users
- **Dashboard** — KPIs, stock status, revenue charts, predictions, pending actions
- **Sales Orders** — Create, edit, filter, search, and export sales
- **Stock Purchases** — Record purchases with wholesale box calculator
- **Customers** — Manage customers with inline order history
- **Suppliers** — Manage suppliers with inline purchase history

### Owner Only
- **Manage Rates** — Update sell/cost prices with change history logging
- **Manage Users** — Create employee accounts, activate/deactivate users
- **Profit & Margin Data** — Visible only to owner across dashboard
- **Delete Records** — Only owner can delete any records

## Business Logic

| Metric | Calculation |
|--------|------------|
| Stock Remaining | Total purchased dz − Total sold dz (per size) |
| Revenue | Sum of amount where payment_status = 'Paid' |
| COGS | Sum of (quantity_dz × cost_price_per_dz) for all sales |
| Gross Profit | Revenue − COGS |
| Gross Margin % | (Gross Profit ÷ Revenue) × 100 |
| Outstanding Dues | Sum of amount where payment = 'Due' or 'Partial' |
| Daily Velocity | Total dz sold (last 7 days) ÷ 7 per size |
| Days Until Stockout | Remaining stock ÷ daily velocity (∞ if velocity = 0) |
| Projected Revenue | Sum of (remaining_dz × sell_price_per_dz) per size |
